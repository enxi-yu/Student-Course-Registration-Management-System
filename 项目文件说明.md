# 学生选课管理系统教师端网站项目文件说明

本文档说明 `D:\Student-Course` 网站项目中各目录和主要文件的作用，方便后续继续开发教师模块，也方便和学生端、管理员端同学协作。

## 一、项目整体架构

当前项目已经改成 B/S 网站架构：

```text
浏览器页面 HTML/CSS/JS
        ↓ HTTP 请求
Controllers API 控制器
        ↓ 调用
Services 业务逻辑层
        ↓ 调用
Repositories 数据访问层
        ↓ SQL
Oracle 数据库
```

也就是说：

- 前端页面负责展示和交互。
- Controller 负责接收浏览器请求。
- Service 负责业务规则判断。
- Repository 负责真正查询或修改数据库。
- Model 负责定义前后端传输的数据结构。

## 二、根目录文件

### `Student-Course.csproj`

项目文件，告诉 .NET 这是一个 ASP.NET Core Web 项目。

主要作用：

- 指定目标框架：`net8.0`
- 设置项目命名空间：`StudentCourse`
- 引入 Oracle 数据库驱动：`Oracle.ManagedDataAccess.Core`

一般情况下不要频繁修改。只有新增 NuGet 包时才需要改。

### `Program.cs`

网站启动入口。

主要作用：

- 注册 Razor Pages。
- 注册 Controller API。
- 注册教师模块需要的 Repository 和 Service。
- 初始化数据库连接配置。
- 临时设置开发用教师登录状态。
- 启用静态文件访问。
- 映射 Controller 和 Razor 页面路由。

当前里面这句：

```csharp
UserSessionContext.UseDevelopmentTeacherSession();
```

表示开发阶段默认模拟一位教师登录。以后做正式登录功能时，这里需要改成真正的登录认证。

### `appsettings.json`

网站配置文件。

主要作用：

- 保存 Oracle 数据库连接字符串。
- 配置日志级别。
- 配置允许访问的主机。

其中：

```json
"OracleConnection": "User Id=course;Password=Course2026;Data Source=47.116.104.63:1521/COURSEPDB;"
```

就是当前项目连接 Oracle 的配置。

### `appsettings.Development.json`

开发环境配置文件。

主要作用：

- 开发阶段显示更详细的错误信息。
- 设置开发环境日志级别。

通常不用改。

## 三、Controllers 目录

Controller 是网站后端接口层，负责接收浏览器请求，并把请求转给 Service。

### `Controllers/SystemController.cs`

系统公共接口。

当前包含：

- `GET /api/system/ping`：测试网站是否正常运行。
- `GET /api/system/database`：测试 Oracle 数据库连接是否正常。
- `POST /api/dev/session/teacher`：开发阶段切换成教师登录状态。
- `POST /api/dev/session/student`：开发阶段切换成学生登录状态。
- `POST /api/auth/logout`：清除当前登录状态。

这个文件属于公共接口，后续学生端、管理员端也可能会用到。

### `Controllers/TeacherController.cs`

教师模块接口。

当前包含：

- `GET /api/teacher/current`：获取当前教师信息。
- `GET /api/teacher/dashboard`：获取教师首页统计信息。
- `GET /api/teacher/courses`：获取教师课表。
- `GET /api/teacher/classes/{classId}/students`：查看某个教学班的学生名单。
- `GET /api/teacher/classes/{classId}/students/export`：导出学生名单 CSV。
- `GET /api/teacher/course-applications`：查看教师自己的开课申请。
- `POST /api/teacher/course-applications`：提交开课申请。
- `GET /api/teacher/classes/{classId}/scores`：查看成绩录入表。
- `POST /api/teacher/scores`：保存单个学生成绩。
- `POST /api/teacher/classes/{classId}/scores/batch`：批量保存成绩。

以后教师端新增功能时，通常先在这里新增 API 地址。

## 四、Models 目录

Model 是数据模型，负责定义数据长什么样。

### `Models/UserSession.cs`

当前登录用户信息。

保存内容包括：

- 用户 ID
- 用户名
- 真实姓名
- 用户类型
- 教师编号
- 职称
- 学院
- 是否已登录

### `Models/TeacherInfo.cs`

教师个人信息模型。

用于 `/api/teacher/current` 返回当前教师资料。

### `Models/TeacherDashboardDto.cs`

教师首页统计数据模型。

用于展示教师首页上的课程数、班级数、学生数、待处理数量等信息。

### `Models/TeacherClassDto.cs`

教师课表 / 教学班信息模型。

用于展示教师当前负责的课程，包括：

- 教学班 ID
- 教学班名称
- 课程名称
- 学期
- 学分
- 上课时间
- 上课地点
- 课程容量
- 已选人数

### `Models/StudentListDto.cs`

学生名单模型。

用于教师查看某个教学班下有哪些学生。

### `Models/CourseApplicationInput.cs`

提交开课申请时，前端传给后端的数据模型。

属于“输入模型”。

### `Models/CourseApplicationDto.cs`

开课申请显示模型。

用于展示教师提交过的开课申请记录。

### `Models/ScoreDto.cs`

成绩显示模型。

用于教师查看或录入学生成绩。

### `Models/ScoreSaveRequest.cs`

保存成绩时，前端传给后端的数据模型。

属于“输入模型”。

## 五、Services 目录

Service 是业务逻辑层，负责判断规则，不直接关心页面怎么显示。

### `Services/UserSessionContext.cs`

当前登录状态管理。

开发阶段用它模拟教师登录。

以后正式系统中，这里应改成基于 Cookie、Session 或 JWT 的登录状态。

### `Services/TeacherService.cs`

教师核心业务服务。

主要负责：

- 获取当前教师信息。
- 获取教师首页数据。
- 获取教师课表。
- 判断当前用户是否是教师。

### `Services/TeacherStudentService.cs`

教师查看学生名单相关业务。

主要负责：

- 检查教师是否有权限访问某个教学班。
- 获取该教学班学生名单。

### `Services/CourseApplicationService.cs`

教师申请开课业务。

主要负责：

- 校验开课申请数据是否合法。
- 提交开课申请。
- 查询自己的开课申请记录。

### `Services/ScoreService.cs`

成绩管理业务。

主要负责：

- 获取成绩录入表。
- 保存单个学生成绩。
- 批量保存成绩。
- 根据分数计算等级、绩点和获得学分。
- 校验教师是否有权限录入对应教学班成绩。

### `Services/ExportService.cs`

导出 CSV 的辅助服务。

目前网站版导出逻辑已经放在 `TeacherController.cs` 里直接返回文件，这个文件暂时不是核心依赖，可以后续整理时再决定是否保留。

## 六、Repositories 目录

Repository 是数据库访问层，只负责 SQL 和 Oracle 数据读写。

### `Repositories/TeacherRepository.cs`

教师相关数据库查询。

主要负责：

- 根据教师编号查询教师信息。
- 根据用户 ID 查询教师信息。
- 查询教师首页统计数据。
- 查询教师课表。

### `Repositories/TeachingClassRepository.cs`

教学班相关数据库查询。

主要负责：

- 判断某个教学班是否属于当前教师。
- 查询课程学分。

### `Repositories/StudentListRepository.cs`

学生名单数据库查询。

主要负责查询某个教学班下的选课学生。

### `Repositories/CourseApplicationRepository.cs`

开课申请数据库访问。

主要负责：

- 插入教师开课申请。
- 查询教师自己的申请记录。

### `Repositories/ScoreRepository.cs`

成绩数据库访问。

主要负责：

- 查询成绩表。
- 判断学生是否选择了某个教学班。
- 查询已有成绩。
- 保存或更新成绩。

## 七、Infrastructure 目录

基础设施层，放数据库连接、通用技术能力。

### `Infrastructure/DbConnectionFactory.cs`

Oracle 数据库连接工厂。

主要负责：

- 从 `appsettings.json` 读取连接字符串。
- 创建 OracleConnection。
- 打开数据库连接。
- 测试数据库连接是否成功。

所有 Repository 最终都会通过它连接数据库。

`wwwroot` 是网站静态资源目录。浏览器可以直接访问这里面的 HTML、CSS、JS。

### `wwwroot/teacher.html`

教师端主页面。

## 十四、当前运行方式

在 `D:\Student-Course` 目录运行：

```powershell
& 'C:\Program Files\dotnet\dotnet.exe' run --project .\Student-Course.csproj --urls http://localhost:5291
```

浏览器访问：

```text
http://localhost:5291/teacher.html
```

如果提示端口被占用，说明已经有网站进程在运行，可以执行：

```powershell
Get-Process dotnet
Stop-Process -Name dotnet
```

然后重新启动。

### 4. 主要业务规则

学生和教师删除采用禁用账号方式，不做物理删除，避免破坏选课、成绩和日志等历史数据。

教学班容量调整必须满足：

```text
新容量 >= 已选人数
```

选课批次时间必须满足：

```text
开始时间 < 结束时间
```

选课批次状态使用：

```text
0 = 未开始
1 = 进行中
2 = 已结束
```

管理员新增、修改、启用、禁用、重置密码、审批、容量调整、登录等操作会写入系统日志。

### 5. 数据库脚本

管理员端补齐脚本为：

```text
admin-oracle19c-schema.sql
```

该脚本按 Oracle 19c 写法补齐管理员端需要的表、字段和基础数据，包括：

```text
"user"
role
permission
role_permission
student
teacher
administrator
selection_batch
system_log
```

脚本同时补齐开课申请表和课程表的兼容字段，并初始化默认管理员账号：

```text
用户名：admin
密码：admin123
```

如果登录时数据库中不存在默认管理员，后端会尝试自动补齐默认管理员账号；但正常部署时仍建议先执行 `admin-oracle19c-schema.sql`。

### 6. 管理员端运行和排错

网站必须通过 ASP.NET Core 后端启动后访问，不能直接双击打开 `wwwroot/admin.html`。

推荐启动命令：

```powershell
& 'C:\Program Files\dotnet\dotnet.exe' .\bin\Debug\net8.0\Student-Course.dll --urls http://localhost:5291
```

启动后先测试：

```text
http://localhost:5291/api/system/ping
http://localhost:5291/api/system/database
```

如果页面所有操作提示 `Failed to fetch`，一般原因是：

```text
1. 后端没有启动
2. 访问的是 file:/// 路径，而不是 http://localhost:5291/admin.html
3. 端口不是 5291，前端请求地址和后端地址不一致
4. 后端启动了，但 Oracle 数据库连接失败
```

如果 `/api/system/database` 返回 Oracle 连接错误，例如 `ORA-50232`，说明应用已经启动，但当前机器到 Oracle 服务器的地址、端口或服务不可用，需要先检查数据库监听、网络、防火墙和连接字符串。

管理员端人工测试建议顺序：

```text
1. 访问 /api/system/ping，确认后端运行
2. 访问 /api/system/database，确认 Oracle 可连接
3. 打开 /admin.html，使用 admin / admin123 登录
4. 测试课程管理和开课申请，确认原已完成功能未受影响
5. 测试学生管理和教师管理的新增、编辑、禁用、启用、重置密码
6. 测试选课批次新增和修改，检查开始时间必须早于结束时间
7. 测试容量调整，确认新容量不能小于已选人数
8. 查看系统日志，确认管理员操作被记录
```
