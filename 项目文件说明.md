# 学生选课管理系统教师端网站项目文件说明

本文档说明 `D:\Student-Course` 网站项目中各目录和主要文件的作用，方便后续继续开发教师模块，也方便和学生端、管理员端同学协作。

## 一、项目整体架构

当前项目已经改成 B/S 网站架构：

```text
浏览器页面 HTML/CSS/JS
        ↓ HTTP 请求
Controllers API 控制器
        ↓ 调用
Services 业务逻辑层
        ↓ 调用
Repositories 数据访问层
        ↓ SQL
Oracle 数据库
```

也就是说：

- 前端页面负责展示和交互。
- Controller 负责接收浏览器请求。
- Service 负责业务规则判断。
- Repository 负责真正查询或修改数据库。
- Model 负责定义前后端传输的数据结构。

## 二、根目录文件

### `Student-Course.csproj`

项目文件，告诉 .NET 这是一个 ASP.NET Core Web 项目。

主要作用：

- 指定目标框架：`net8.0`
- 设置项目命名空间：`StudentCourse`
- 引入 Oracle 数据库驱动：`Oracle.ManagedDataAccess.Core`

一般情况下不要频繁修改。只有新增 NuGet 包时才需要改。

### `Program.cs`

网站启动入口。

主要作用：

- 注册 Razor Pages。
- 注册 Controller API。
- 注册教师模块需要的 Repository 和 Service。
- 初始化数据库连接配置。
- 临时设置开发用教师登录状态。
- 启用静态文件访问。
- 映射 Controller 和 Razor 页面路由。

当前里面这句：

```csharp
UserSessionContext.UseDevelopmentTeacherSession();
```

表示开发阶段默认模拟一位教师登录。以后做正式登录功能时，这里需要改成真正的登录认证。

### `appsettings.json`

网站配置文件。

主要作用：

- 保存 Oracle 数据库连接字符串。
- 配置日志级别。
- 配置允许访问的主机。

其中：

```json
"OracleConnection": "User Id=course;Password=Course2026;Data Source=47.116.104.63:1521/COURSEPDB;"
```

就是当前项目连接 Oracle 的配置。

### `appsettings.Development.json`

开发环境配置文件。

主要作用：

- 开发阶段显示更详细的错误信息。
- 设置开发环境日志级别。

通常不用改。

## 三、Controllers 目录

Controller 是网站后端接口层，负责接收浏览器请求，并把请求转给 Service。

### `Controllers/SystemController.cs`

系统公共接口。

当前包含：

- `GET /api/system/ping`：测试网站是否正常运行。
- `GET /api/system/database`：测试 Oracle 数据库连接是否正常。
- `POST /api/dev/session/teacher`：开发阶段切换成教师登录状态。
- `POST /api/dev/session/student`：开发阶段切换成学生登录状态。
- `POST /api/auth/logout`：清除当前登录状态。

这个文件属于公共接口，后续学生端、管理员端也可能会用到。

### `Controllers/TeacherController.cs`

教师模块接口。

当前包含：

- `GET /api/teacher/current`：获取当前教师信息。
- `GET /api/teacher/dashboard`：获取教师首页统计信息。
- `GET /api/teacher/courses`：获取教师课表。
- `GET /api/teacher/classes/{classId}/students`：查看某个教学班的学生名单。
- `GET /api/teacher/classes/{classId}/students/export`：导出学生名单 CSV。
- `GET /api/teacher/course-applications`：查看教师自己的开课申请。
- `POST /api/teacher/course-applications`：提交开课申请。
- `GET /api/teacher/classes/{classId}/scores`：查看成绩录入表。
- `POST /api/teacher/scores`：保存单个学生成绩。
- `POST /api/teacher/classes/{classId}/scores/batch`：批量保存成绩。

以后教师端新增功能时，通常先在这里新增 API 地址。

## 四、Models 目录

Model 是数据模型，负责定义数据长什么样。

### `Models/UserSession.cs`

当前登录用户信息。

保存内容包括：

- 用户 ID
- 用户名
- 真实姓名
- 用户类型
- 教师编号
- 职称
- 学院
- 是否已登录

### `Models/TeacherInfo.cs`

教师个人信息模型。

用于 `/api/teacher/current` 返回当前教师资料。

### `Models/TeacherDashboardDto.cs`

教师首页统计数据模型。

用于展示教师首页上的课程数、班级数、学生数、待处理数量等信息。

### `Models/TeacherClassDto.cs`

教师课表 / 教学班信息模型。

用于展示教师当前负责的课程，包括：

- 教学班 ID
- 教学班名称
- 课程名称
- 学期
- 学分
- 上课时间
- 上课地点
- 课程容量
- 已选人数

### `Models/StudentListDto.cs`

学生名单模型。

用于教师查看某个教学班下有哪些学生。

### `Models/CourseApplicationInput.cs`

提交开课申请时，前端传给后端的数据模型。

属于“输入模型”。

### `Models/CourseApplicationDto.cs`

开课申请显示模型。

用于展示教师提交过的开课申请记录。

### `Models/ScoreDto.cs`

成绩显示模型。

用于教师查看或录入学生成绩。

### `Models/ScoreSaveRequest.cs`

保存成绩时，前端传给后端的数据模型。

属于“输入模型”。

## 五、Services 目录

Service 是业务逻辑层，负责判断规则，不直接关心页面怎么显示。

### `Services/UserSessionContext.cs`

当前登录状态管理。

开发阶段用它模拟教师登录。

以后正式系统中，这里应改成基于 Cookie、Session 或 JWT 的登录状态。

### `Services/TeacherService.cs`

教师核心业务服务。

主要负责：

- 获取当前教师信息。
- 获取教师首页数据。
- 获取教师课表。
- 判断当前用户是否是教师。

### `Services/TeacherStudentService.cs`

教师查看学生名单相关业务。

主要负责：

- 检查教师是否有权限访问某个教学班。
- 获取该教学班学生名单。

### `Services/CourseApplicationService.cs`

教师申请开课业务。

主要负责：

- 校验开课申请数据是否合法。
- 提交开课申请。
- 查询自己的开课申请记录。

### `Services/ScoreService.cs`

成绩管理业务。

主要负责：

- 获取成绩录入表。
- 保存单个学生成绩。
- 批量保存成绩。
- 根据分数计算等级、绩点和获得学分。
- 校验教师是否有权限录入对应教学班成绩。

### `Services/ExportService.cs`

导出 CSV 的辅助服务。

目前网站版导出逻辑已经放在 `TeacherController.cs` 里直接返回文件，这个文件暂时不是核心依赖，可以后续整理时再决定是否保留。

## 六、Repositories 目录

Repository 是数据库访问层，只负责 SQL 和 Oracle 数据读写。

### `Repositories/TeacherRepository.cs`

教师相关数据库查询。

主要负责：

- 根据教师编号查询教师信息。
- 根据用户 ID 查询教师信息。
- 查询教师首页统计数据。
- 查询教师课表。

### `Repositories/TeachingClassRepository.cs`

教学班相关数据库查询。

主要负责：

- 判断某个教学班是否属于当前教师。
- 查询课程学分。

### `Repositories/StudentListRepository.cs`

学生名单数据库查询。

主要负责查询某个教学班下的选课学生。

### `Repositories/CourseApplicationRepository.cs`

开课申请数据库访问。

主要负责：

- 插入教师开课申请。
- 查询教师自己的申请记录。

### `Repositories/ScoreRepository.cs`

成绩数据库访问。

主要负责：

- 查询成绩表。
- 判断学生是否选择了某个教学班。
- 查询已有成绩。
- 保存或更新成绩。

## 七、Infrastructure 目录

基础设施层，放数据库连接、通用技术能力。

### `Infrastructure/DbConnectionFactory.cs`

Oracle 数据库连接工厂。

主要负责：

- 从 `appsettings.json` 读取连接字符串。
- 创建 OracleConnection。
- 打开数据库连接。
- 测试数据库连接是否成功。

所有 Repository 最终都会通过它连接数据库。

## 八、wwwroot 目录

`wwwroot` 是网站静态资源目录。浏览器可以直接访问这里面的 HTML、CSS、JS。

### `wwwroot/teacher.html`

教师端主页面。

这是教师登录后看到的主界面，包含教师模块的页面结构。

### `wwwroot/index.html`

静态首页。

当前项目入口主要会跳转或进入教师端页面。

### `wwwroot/css/app.css`

通用样式。

适合放各模块都能复用的基础页面样式。

### `wwwroot/css/teacher.css`

教师端专用样式。

负责教师首页、课表、申请开课、学生名单、成绩录入等页面的视觉效果。

### `wwwroot/js/api.js`

前端 API 请求封装。

这是前端和后端连接的关键文件。

当前网站版会通过 `fetch` 请求这些接口：

- `/api/teacher/current`
- `/api/teacher/dashboard`
- `/api/teacher/courses`
- `/api/teacher/course-applications`
- `/api/teacher/classes/{classId}/students`
- `/api/teacher/classes/{classId}/scores`

以后前端页面不要直接到处写 `fetch`，尽量统一通过这个文件请求后端。

### `wwwroot/js/app.js`

系统入口脚本。

负责首页或通用页面的初始化逻辑。

### `wwwroot/js/teacher-app.js`

教师端主脚本。

主要负责：

- 初始化教师端页面。
- 管理菜单切换。
- 调用各个页面模块。
- 加载教师信息。

### `wwwroot/js/pages/teacher-dashboard.js`

教师首页页面逻辑。

负责加载并展示教师首页统计信息。

### `wwwroot/js/pages/teacher-courses.js`

教师课表查看页面逻辑。

负责调用 `/api/teacher/courses`，并把教师课程显示在页面上。

### `wwwroot/js/pages/class-students.js`

学生名单页面逻辑。

负责查看某个教学班的学生列表。

### `wwwroot/js/pages/course-application.js`

申请开课页面逻辑。

负责提交开课申请、展示申请记录。

### `wwwroot/js/pages/score-entry.js`

成绩录入页面逻辑。

负责查看学生成绩、保存单个成绩、批量保存成绩。

### `wwwroot/lib`

Visual Studio 模板自带的前端库目录。

里面包括：

- Bootstrap
- jQuery
- jQuery Validation

如果当前教师端页面没有用到这些库，可以暂时不管。不要随便删除，除非确认项目完全不依赖。

## 九、Pages 目录

这是 Razor Pages 模板自带目录。

### `Pages/Index.cshtml`

网站默认首页。

当前已经改成自动跳转到 `/teacher.html`。

### `Pages/Error.cshtml`

网站错误页。

当程序出现未处理异常时会显示。

### `Pages/Privacy.cshtml`

模板自带隐私页。

当前业务基本用不到，可以保留，也可以后续删除。

### `Pages/Shared/_Layout.cshtml`

Razor Pages 的布局页。

当前教师端主要使用 `wwwroot/teacher.html`，所以这个布局页对教师端影响不大。

## 十、Properties 目录

### `Properties/launchSettings.json`

Visual Studio 启动配置。

里面定义了网站运行时的地址，例如：

```text
http://localhost:5291
https://localhost:7186
```

如果遇到端口占用，可以在这里修改端口，或者关闭旧的 dotnet 进程。

## 十一、bin 和 obj 目录

这两个目录是编译生成目录。

### `bin`

保存编译后的程序文件。

### `obj`

保存编译过程中的中间文件。

这两个目录不要手动写业务代码，也不要把它们当成主要开发文件。

wwwroot/css/teacher.css

wwwroot/js/pages/teacher-courses.js

```text
wwwroot/js/pages/course-application.js
```

```text

```

### 新增业务规则

改或新增：

```text
Services/*.cs
```

### 新增数据库查询

改或新增：

```text
Repositories/*.cs
```

### 新增前后端传输数据结构

改或新增：

```text
Models/*.cs
```

## 十三、学生端和管理员端如何扩展

建议后续按模块拆分：

```text
Controllers/TeacherController.cs
Controllers/StudentController.cs
Controllers/AdminController.cs

Services/TeacherService.cs
Services/StudentService.cs
Services/AdminService.cs

Repositories/TeacherRepository.cs
Repositories/StudentRepository.cs
Repositories/AdminRepository.cs

wwwroot/teacher.html
wwwroot/student.html
wwwroot/admin.html
```

公共文件继续复用：

```text
Infrastructure/DbConnectionFactory.cs
Models/UserSession.cs
wwwroot/js/api.js
wwwroot/css/app.css
```

这样每个人负责自己的模块，同时不会互相乱改。

## 十四、当前运行方式

在 `D:\Student-Course` 目录运行：

```powershell
& 'C:\Program Files\dotnet\dotnet.exe' run --project .\Student-Course.csproj --urls http://localhost:5291
```

浏览器访问：

```text
http://localhost:5291/teacher.html
```

如果提示端口被占用，说明已经有网站进程在运行，可以执行：

```powershell
Get-Process dotnet
Stop-Process -Name dotnet
```

然后重新启动。
